A Pixel is not a pixel

- Peter-Paul Koch - Twitter: @ppk - Video: http://vimeo.com/52851511

This was a fast paced and interesting presentation. I've learned a lot about pixels and where you can run into presenation issues on devices. I've heard of Peter-Paul Koch but never seen or attended his presentations. It was a great experience and worth while!

Let's start:

You have 3 kinds of pixels:

  • css: like width:20px. Their size may be increased or decreased.
  • device: The actual pixels on the screen. When the number of device pixels increases would break sites, so density-independent pixels were invented.
  • densitiy independent

Device pixels are the physical pixels on the the device. But they get higher and higher pixel densities, and the number of devic pixels increases.

When building sites for different media/resolutions, ask your self this question: What kind of pixels am I using? CSS pixels related to javascript or device pixels when used in the code? In Javascript pixels might not be CSS pixels all the time. Usually it's CSS pixel, but sometimes it might be device pixels (dips) when related to screen size.

What is a viewport?

It is the total amount of space available for css layouts:

  • On desktop it's equal to the browser window. An HTML element has an implicit width: 100% and spans the entire viewport.

Viewport for mobile

The viewport for mobile is quite a bit more complicated. If the narrow browser window were to be the viewport, many sites would be squeezed to death. And mobile browsers must render all sites correctly even if they haven't been mobile optimized.

“On Retina displays, device pixels are 640px, but device-independent pixels are still 320px to not break the Web on mobile” - @ppk

To work around this the mobile browser vendors split viewport in 2:

  • layout viewport: css
  • visual viewport: actually what you use/see.

You can compare it with looking through a keyhole. If you want to know how much of the website the user is actually seeing, use window.innerWidth (works on virtually all browsers except IE9 and Android 2.0)

What does zooming do with the viewport?

  • on desktop the viewport stays the same, the css pixels become larger and less.
  • mobile visual viewport becomes smaller but the layout viewport remains static, thus css declarations are not recomputed. The visual viewport now contains less css pixels.

Behavior of position:fixed

What if you use/view position:fixed on the mobile? And what is fixed: the viewport. Ok, but which viewport? The layout viewport or the visual view?

It actually is browser dependent. There are differences in what browsers see as viewport:

  • Android, Chrome and Blackberry browsers use the visual viewport.
  • IE9 the layout viewport.
  • Opera "hops".
  • And safari does something weird.
  • On layout viewport position: fixed behaves as absolute.
  • On Opera the fixed hops up and down.

Use these demo pages on your device/browsers to see the difference in behavior: http://quirksmode.org/m/tests/positionfixed.html And http://quirksmode.org/m/tests/devicefixed.html

You can use the javascript:pageOffset, it works the same on mobile and desktop.

Media queries and Meta viewport

Media queries

When using media queries, use width, and not device-width. Device-width is the width of your screen, that is not interesting.

Width: Gives the width of the layout viewport and is in css pixels

Meta viewport

The meta name viewport tag tells the browser to set the size of the layout viewport. With this tag you can give a pixel value, or a device-width.

The trick for responsive design: use the meta name viewport width="device-width" and media queries.
